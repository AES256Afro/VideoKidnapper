# VideoKidnapper v1.2.0

*Released 2026-04-18*

The biggest release since v1.0. Three themes: **a real editor** (undo/redo, audio playback, captions, color grade, crossfades, snap guides), **a batch mode** (queue many files, queue many URLs, queue many clips of one video), and **real distribution** (PyPI, Windows installer, Homebrew, winget).

## What's new

### Playback & editing

- **Real-time playback with sound.** The Play button now streams video + audio in lock-step through a dedicated decoder — no more silent 8-fps frame scrub. Optional deps (`imageio-ffmpeg`, `sounddevice`); installs without them fall back gracefully.
- **Undo / redo everywhere.** `Ctrl+Z` / `Ctrl+Y` / `Ctrl+Shift+Z` across text-layer edits, image overlays, crop rectangle, trim range, and queued ranges. 50-step history, debounced so typing a sentence collapses into one undo entry.
- **Thumbnail strip** above the waveform. 32 frames pulled in the background, scrubbable with a click. The active trim range is highlighted; outside it dims.
- **Snap-to-guides** when dragging text layers — snaps to frame center, padded edges, and every edge + center of every other layer. Dashed guide lines appear while snapping, clear on release.
- **Color grade sliders** in Export Options — Brightness, Contrast, Saturation, Gamma. One-click **Reset** returns to neutral. Neutral values compile to no filter at all, so "I don't want a color tweak" stays free.
- **Image overlays.** A new panel under Trim lets you drop in PNG / JPG / WebP / GIF / BMP watermarks, reaction stickers, or brand bugs — with anchor, scale, opacity, and start/end timing. Live preview on the scrub canvas; GIF export supported via an automatic intermediate-MP4 palette-pass.
- **Transitions between queued ranges** — Cut (the existing fast concat), Crossfade, Fade to black, Fade to white. Duration picker: 0.25s / 0.5s / 1s / 1.5s. Turns "concat these 5 trims" into "make a compilation."
- **🗣 Auto-captions via Whisper.** New button in the Trim tab runs [faster-whisper](https://github.com/guillaumekln/faster-whisper) over the current range and imports the result through the SRT pipeline. Pick model size (tiny / base / small / medium / large) in a small dialog. Optional dep — button hints `pip install faster-whisper` when missing.

### Export presets & batch modes

- **Platform export presets** on the Trim tab: **YouTube 1080p / Shorts**, **Instagram Reel / Post / Story**, **TikTok**, **Twitter / X**, **Bluesky**, **Discord 8 MB / 25 MB**, **Slack GIF**, **Web Embed**, plus **Custom**. One click sets Quality + Format + aspect; editing any of those three flips the label back to Custom so the dropdown never lies about what's selected.
- **⎆ Batch Export tab.** Queue many local files, apply shared Export Options, hit Start. Sequential encode in a background thread with per-row live status, Stop button that interrupts cleanly, and per-row **platform override** (default "Inherit" takes the batch-wide settings). The queue, overrides, and terminal statuses survive app restarts — a force-quit mid-encode resumes with the in-flight row normalised to queued.
- **Right-click menu on batch rows** — Reveal output in Explorer, Open source folder, Reset status to queued, Remove.

### UI & discoverability

- **Keyboard shortcuts overlay.** Press `?` anywhere — or click the new `⌨` chip in the header — to open a categorized cheat sheet of every binding (playback, trim, edit, file & export, help). Esc or a second `?` dismisses. `?` inside a text entry still types a literal `?`.

### Plugin system

- **Third-party plugins.** A plugin is now a regular pip-installable package that declares itself under the `videokidnapper.plugins` entry-point group. VideoKidnapper discovers it at startup and calls `on_app_ready(app)`, from which it can call `app.register_tab(...)`, log to the Debug tab, or post to the status bar. Plugins declare `min_app_version` / `max_app_version` and incompatible ones are skipped with a clear reason. One broken plugin never kills the app.
- **Example plugin** at `examples/plugins/videokidnapper_hello/` + developer docs at [docs/PLUGINS.md](../PLUGINS.md). Plugins may adopt any license — the entry-point mechanism is packaging-level, not source-level.

### Distribution

- **PyPI package.** `pip install videokidnapper` (core) or `pip install "videokidnapper[dnd]"` (+ drag-and-drop). Installs a `videokidnapper` console script that launches the GUI, or the CLI with any flag.
- **Standalone Windows `.exe`.** Download `VideoKidnapper.exe` from the GitHub Release, double-click, done — no Python install required. Built via PyInstaller on tag push.
- **Inno Setup installer** (`VideoKidnapper-Setup.exe`) — traditional Windows wizard with Start Menu entries, Programs & Features registration, and clean uninstall. Sits alongside the portable `.exe` for users who prefer a real installer.
- **Homebrew tap** at `AES256Afro/homebrew-videokidnapper` — `brew install aes256afro/videokidnapper/videokidnapper`.
- **winget manifest** — `winget install AES256Afro.VideoKidnapper`.

### Reliability

- **Tier-1 robustness pass.** The ffprobe crash on missing streams, yt-dlp hanging on dead URLs, two export threads racing each other's settings writes, and Ctrl+V pasting into the URL tab when focus is elsewhere — all fixed.
- **`ProbeError`** exception type — narrow, catchable, user-surfacing — replacing bare `subprocess.CalledProcessError` leakage.
- **Atomic settings writes.** A crash mid-write never leaves the JSON file truncated.

### Under the hood

- `ffmpeg_backend.py` split from a 1200-line monolith into a focused `ffmpeg/` subpackage (probe, filters, encode, concat, \_internals). The old import path still works — existing callers and tests didn't change a line.
- **mypy in CI** with a per-module strict allowlist. Seven utility modules are strict-clean; the rest default to non-strict until typed.
- **Integration test that actually runs ffmpeg** — builds a 1-second synthetic video via lavfi, runs the full `trim_to_video` path, skips when ffmpeg isn't on PATH. Catches wiring regressions unit tests can't see.
- **Settings schema v4** (v1 → v2 → v3 → v4, additive) — persists color grade, concat transitions, and the Batch Export queue.

## Install

| Channel | Command / link |
|---|---|
| PyPI | `pip install videokidnapper` |
| Windows portable | `VideoKidnapper.exe` on the [Release page](https://github.com/AES256Afro/VideoKidnapper/releases/tag/v1.2.0) |
| Windows installer | `VideoKidnapper-Setup.exe` on the same page |
| Homebrew | `brew install aes256afro/videokidnapper/videokidnapper` |
| winget | `winget install AES256Afro.VideoKidnapper` |

FFmpeg remains an external prerequisite on every channel. The in-app **⚙ Setup** dialog handles portable FFmpeg install on Windows; macOS / Linux users install via their package manager.

## Upgrading from v1.1.0

Nothing breaks. The settings schema migration is additive. Optional deps (`imageio-ffmpeg`, `sounddevice`, `faster-whisper`) unlock playback-with-audio and auto-captions, but the app runs fine without them.
