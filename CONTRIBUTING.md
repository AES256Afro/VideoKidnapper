# Contributing

Thanks for considering a contribution. This is a small project and issues / PRs are welcome.

## Getting started

```bash
git clone https://github.com/AES256Afro/VideoKidnapper.git
cd VideoKidnapper
pip install -r requirements.txt
pip install pytest
```

Run the test suite before you change anything so you know what "green" looks like:

```bash
python -m pytest tests/ -q
```

The app boots with:

```bash
python main.py           # GUI
python main.py --help    # CLI flags
```

The full prerequisite installer runs from **⚙ Setup** inside the GUI — use it on a fresh machine to verify everything still auto-installs cleanly.

## Coding conventions

- 4-space indent, no tabs. Snake case for identifiers.
- Prefer the project's **theme tokens** (`T.ACCENT`, `T.BG_SURFACE`, etc.) over hard-coded hex.
- Use `T.font(...)` and `T.button(...)` factories so widgets stay consistent.
- **Escape everything** that flows into an ffmpeg filter argument through `utils/ffmpeg_escape.py`. Never format user text directly into a filter string.
- Add a test if you change filter construction, URL detection, escape logic, settings schema, or any pure helper.
- GUI-only changes don't need tests, but please verify `python main.py` still boots and the affected tab renders.

## Tests

The suite is pure-logic and runs in under a second on a laptop. It intentionally avoids network and subprocess calls — `test_github_update.py` tests parsing only, not the HTTP path.

```bash
python -m pytest tests/ -v
```

If you add a feature that has a meaningful pure surface, add a file under `tests/test_<feature>.py`.

## Regenerating screenshots

The README images are generated by `scripts/capture_screenshots.py`, which drives the app through each state and grabs the window. To regenerate:

```bash
ffmpeg -y -f lavfi -i "testsrc=duration=6:size=1280x720:rate=24" \
       -f lavfi -i "sine=frequency=440:duration=6" \
       -c:v libx264 -c:a aac -pix_fmt yuv420p /tmp/vkshots/demo.mp4

VK_DEMO_VIDEO=/tmp/vkshots/demo.mp4 python scripts/capture_screenshots.py
```

## Submitting a PR

1. Fork, branch off `main` with a feature-style name (`feat/…`, `fix/…`, `docs/…`).
2. Keep commits focused. Use a descriptive subject line.
3. Run `python -m pytest tests/ -q` and confirm all green.
4. Open the PR against `main` with a short **Summary** and a **Test plan** checklist.

## Reporting bugs

Open a GitHub issue with:
- OS + Python version
- Steps to reproduce
- The full traceback or ffmpeg error from the **Debug** tab (the Debug tab captures stderr output including ffmpeg failures)
- If the issue involves a URL, whether it happens with public and private content
